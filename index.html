Estabelecendo conexões SSH com o GitHub

Etapas do processo

Para realizar a conexão utilizando SSH, crie localmente um par de chaves (chave-pública e chave-privada), e configure no GitHub a chave privada. Assim a máquina local será reconhecida e aceita como confiável.

O local para inserção da chave pública no GitHub está localizado em "SSH and GPG Keys", na URL github.com/settings/keys.

Para gerar a chave, através do terminal utilize a linha de comando:

$ ssh-keygen -t ed25519 -C "seuemail"

Após o retorno do comando, confirme a localização e informe uma senha para a sua chave. Um par de chaves será gerado em "/home/Usuario/.ssh/id_25519".

Navegue até a localização da chave para resgatar a chave pública com a linha de comando:

$ cat id_ed25519.pub

Copie a chave exibida e informe-a ao GitHub, na seção descrita no início do processo.

(!) Agora o GitHub já possui a chave pública do computador local, mas é necessário inicializar as chaves (localmente) com o SSH Agent. Para realizar a inicialização é preciso seguir as etapas:

$ eval $(ssh-agent -s)

Será exibido um retorno como: "Agent pid 1111".

Agora será necessário entregar a chave privada ao SSH Agent, com o comando:

$ ssh-add id_ed25519, informando a senha para prosseguir.
